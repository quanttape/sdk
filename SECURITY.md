# Security Policy

Security is the core focus of Quant Tape. We take all vulnerabilities and security-related bugs in this SDK seriously.

## Supported Versions

Currently, only the latest release of the SDK receives security updates. If you are using an older version, please upgrade to the latest version before reporting a bug, as it may have already been resolved.

| Version | Supported          |
| ------- | ------------------ |
| >= 0.0.15| :white_check_mark: |
| < 0.0.15 | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you have found a security vulnerability in the Quant Tape SDK, please report it privately to our team.

1. **Email:** Send your report to `info@quanttape.com`.
2. **Subject Line:** Please include "[VULNERABILITY]" in the subject line to ensure it is flagged for immediate review.
3. **Details:** Provide a clear description of the issue, including steps to reproduce it, the operating system, and the version of Python and the SDK you are using. If applicable, please attach a minimal reproducible example or proof-of-concept code.

## Disclosure Policy

* We will acknowledge receipt of your vulnerability report within 48 hours.
* We will send you regular updates about our progress in investigating and resolving the issue.
* Once the issue is resolved, we will publish a security advisory and notify users to upgrade. We ask that you maintain strict confidentiality until the patch is publicly available.
